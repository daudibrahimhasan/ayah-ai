Section 1 — Product Requirements Document (PRD)
1.1  App Overview
Ayah (آية) is an AI-powered Quran Ayat recognition mobile application for Android. The name means 'verse' or 'sign' — one of the most fundamental concepts in Quranic study. Ayah listens to any live or recorded Quranic recitation and instantly identifies the Surah and Ayat being recited, then allows the user to tap the result to open the full Quran PDF at the exact matching Surah page — completely offline, with no internet connection required.
1.2  Problem Statement
Muslims worldwide — especially during Tarawih prayers — often hear beautiful Quranic recitations and cannot identify the exact Surah or Ayat being recited. Existing Quran apps require manual searching. No current app provides both:
Real-time Ayat recognition for any unknown reciter in a live mosque environment
Instant navigation to the identified Ayat inside a full Quran PDF for deeper reading and context

Section 2 — Functional Requirements Document (FRD)
2.1  Feature: Ayat Recognition
User Flow
User opens app and lands on the Home screen
User taps the large mint microphone button at the centre of the screen
App requests microphone permission (first time only)
App begins listening — waveform animation plays around the button
After 3–10 seconds, app processes the captured recitation on-device
Result card slides up: Surah name (Arabic + transliteration), Ayat number, Arabic text, English translation, match confidence score

User can also bookmark the result or dismiss it

Section 3 — Technical Specification Document
3.1  Architecture Overview
Ayah uses an offline-first architecture. The AI model, Quran database, Quran PDF, and Surah-to-page mapping are all bundled inside the APK. No internet is required for any core feature. An optional online mode uses a hosted model for higher accuracy when connectivity is available.
ℹ  Full Flow: Mic Input → Audio Preprocessing → Whisper Model → Arabic Text → Fuzzy Match → Quran DB → Result Card → [User Taps] → PDF Viewer opens at mapped Surah page

3.5  Audio Processing Pipeline
Capture: 16kHz, mono, 16-bit PCM via device microphone
Chunk: Split into 30-second windows (Whisper maximum input length)
Normalise: Adjust volume levels for consistent model input
Denoise: Basic noise gate to reduce mosque reverb and crowd noise
Convert: Feed normalised float32 array into Whisper ONNX model
Post-process: Strip diacritics for fuzzy matching if needed

Section 5 — API Documentation
ℹ  Ayah is primarily an offline app. The PDF viewer, Quran browser, and all local features require no API. The API is only used when the user has internet access and the app uses online mode for higher-accuracy recognition. The backend is hosted on Hugging Face Spaces (free tier).
5.1  Base URL
https://ayah-api.hf.space/api/v1
5.2  Authentication
Version 1.0 uses no authentication. The API is public and rate-limited by IP. Future versions may introduce optional API keys for higher-tier usage.

5.4  App Integration Flow
App checks connectivity on each recognition attempt
Online: POST audio to /recognize → get result with pdf_page → show result card
Offline: run local Whisper → match local DB → look up pdf_page from local JSON
User taps 'Open in PDF' → app calls pdfController.jumpToPage(pdf_page)
PDF opens instantly — no network call needed regardless of online/offline mode
On API error or timeout: seamlessly fall back to offline recognition
